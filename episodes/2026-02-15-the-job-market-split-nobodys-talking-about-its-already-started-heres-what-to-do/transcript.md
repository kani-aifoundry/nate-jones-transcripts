# The Job Market Split Nobody's Talking About (It's Already Started). Here's What to Do About It.

Code is about to cost nothing, and knowing what to build is about to cost you everything. Last July, an AI coding agent deleted Saster's entire production database during an explicit code freeze and then fabricated thousands of fake records to cover its tracks. Jason Linen was a developer who had given the agent all caps instructions. I guess that's how we prompt, not to make any changes. The agent made changes anyway, destroyed the data, and lied about it during a code freeze. Fortune covered this. The register covered it. It made headlines because an agent ignored a clear spec. But that failure is what people fixate on incorrectly. It's the story of the disobedient machine, the Terminator. The failure mode that actually matters is quieter and it's vastly more expensive. Agents that execute specifications flawlessly, they build exactly what was asked for and then what was asked for is wrong. A code rabbit analysis of 470 GitHub pull requests found AI generated code produces 1.7 times more logic issues than human written code. Not syntax errors, not formatting problems, but the code itself doing the wrong thing. tracked a 9% climb in bug rates that correlates to the 90% increase in AI adoption alongside a 91% increase in code review time. The code ships faster, but it's often more wrong and it's difficult to catch until production. AWS now has a developer environment whose core innovation isn't faster code generation. It's actually just forcing developers to write a testable specification before any code gets generated. Tell me what it's going to be like by telling me how you test it. Amazon, a company that profits when you ship faster, decided the most valuable thing it could do is slow you down and define what you want because error rates were that concerning when developers did not write tests. This tells you everything about where the bottleneck in code is moving in the market.

[AWS Cairo: Forcing Specification Before Code]
Specification is the new bottleneck. When the cost of production goes to zero, the value moves to specification. We are seeing this split between people who can specify what they want and people who just push buttons.

[Chollet's Framework and Why It's Incomplete]
François Chollet has a great framework for intelligence, but it often misses the practical application of how agents interact with complex systems. The ability to generalize is key, but so is the ability to maintain state and intent across long horizons.

[When Cost Goes to Zero, Demand Goes Infinite]
Jevons Paradox states that as something becomes more efficient, we use more of it. This is happening with code. We are going to see an explosion of disposable software, but only if we can specify it correctly.

[The Specification Bottleneck]
The real cost now isn't the code; it's the cost of a wrong specification. If an agent builds exactly what you asked for, but you asked for the wrong thing, you've wasted compute and time.

[Two Classes of Knowledge Workers Emerging]
There's a 10-80x revenue gap between those who can leverage AI as a force multiplier for their intent and those who are just being replaced by it. The "High-V" (High Value) tokens are the ones that define the 'what' and 'why'.

[The Solopreneur Thesis: Who It Actually Serves]
AI is supercharging the individual. One person can now do the work of a team, but only if they have the system-level thinking to manage the agents.

[Knowledge Work Is Converging on Software]
Every job is becoming a software job. Whether you're in legal, marketing, or HR, you are essentially writing specifications for agents to execute.

[Hit the Right Level of Abstraction]
Don't get bogged down in the syntax. Move up the stack. Focus on the business logic and the outcomes.

[Learn to Work With Compute, Not Just About It]
Understand how these models think. They aren't humans; they are statistical engines that follow instructions literally.

[Think in Systems, Not Documents]
A specification isn't a static Word doc; it's a living system of tests and constraints.

[Audit Your Role for Coordination Overhead]
If your job is mostly just passing information between people, you're at risk. You need to move to the edges where the value is created through specification.

[The J-Curve Trough We're In]
We are in a period of transition where traditional SAS stocks are being disrupted. Claude was able to wipe off billions from traditional stocks by releasing a simple prompter for legal work. This is the new reality. Between engineers who can drive high-V value tokens is literally $285 billion. The point is to think about knowledge workers and understand that we need to have a much more rigorous approach to how we define the work we want done. If we don't, we're just building the wrong things faster.
