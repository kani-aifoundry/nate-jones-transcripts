# I Built an 11-Tab Financial Model in 10 Minutes. The $20/Month Tool That's About Change How We Work.

Claude is invading Excel and they are taking no prisoners. I built an 11 tab financial model of building versus renting a home across every zip code in the country in just 10 minutes. The time it takes to make a good cup of coffee. And if you're wondering, yes, it was that detailed. It had a sensitivity analysis. It had an opportunity cost comparison versus S&P 500 returns. I could go on and on. The point is actually not the detail of that particular spreadsheet, which I will be showing you a little bit later in this video. The point is that I truly partnered with Claude to get that done in a way that I have not been able to partner with any AI before. Claude itself suggested some of these tabs and was then able to go get the data and able to build them out. This is not incremental improvement. This is a phase change in what knowledge work really means and I'm here to introduce it to you. But first, we should understand how strange it is to have Claude in Excel.

Foundation models, whether we're talking about Claude or GPT or Llama, have mostly existed inside the browser in a special chat interface. And if you want to use the output of that model in a specialized tool like Excel, you have to do the manual work of jumping from tab to tab, copying and pasting the result, and that's slow. It's high friction. And for most professional use cases, it's just not worth it. But something changed a few weeks ago. Anthropic released an official add-in that puts Claude directly into Excel. Now you can use Claude like any other Excel formula. You give it a piece of information and tell it what to do, and it does it right there in the cell.

This seems like a small UI change, but it's actually much bigger than that. When Claude is in Excel, it has access to all the data that's in your spreadsheet. It can see the numbers, it can see the labels, it can see the structure. And that means it can do things that a regular chat interface just can't do. It can help you clean data, it can help you categorize things, it can even help you build entire models from scratch. And that's exactly what I did with this rent versus buy model.

I didn't start with a blank spreadsheet and ask Claude to fill it in. I started with a blank spreadsheet and had a conversation with Claude inside Excel. I told it what I wanted to build—a model that would compare the cost of renting versus buying a home in different zip codes across the country. And then Claude started suggesting what kind of data we would need and how we should structure the tabs. It suggested a tab for mortgage rates, a tab for property taxes, a tab for historical rent prices, and so on. And then, and this is the really crazy part, it went and got the data.

Because Claude is connected to the internet through its API, it can pull in real-time information. It pulled in current mortgage rates, it pulled in property tax averages for different states, and it even pulled in some historical rent data. Now, to be clear, it's not perfect. Sometimes it gets a number wrong, or it pulls from a source that's a little bit out of date. You still have to be the pilot here. You still have to check its work. But the speed of transformation and not be clear about where I see some bumps along the way. First, what's transformative? The core capability is genuinely weeks to minutes fast. You can go to building, explaining, debugging, and iterating in just a few minutes on a spreadsheet that previously might have taken you into the weeks, especially fetching the data. Claude can handle multi-tab reasoning in a way that I have not been able to do with any other tool.

But there are bumps. The moat has to come from somewhere else. Anthropic's answer is the competitive mode is workflow integration backed by data partnerships. So don't just build a better model, embed it where work happens and connect it to information that competitors have trouble accessing. Excel was a very obvious first target. Not because spreadsheets are particularly sexy, but because Excel is the operational nervous system of the global economy.

Many finance teams really hate this because they want control over when work gets saved and the ability to experiment without making permanent changes.
